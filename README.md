# IDBI MSME HealthLens

Bank-grade internal web app for IDBI credit officers — an explainable **MSME Financial Health Card** built from consented alternate data (GST, Account Aggregator bank, UPI, EPFO, **electricity / power consumption**, bureau-lite). Built for IDBI Innovate 2026, Track 03 (Financial Inclusion / Digital Lending / Credit Decisioning).

> HealthLens is a decision-support and configurable recommendation engine. IDBI remains the regulated entity and final decision owner. All data shown is synthetic prototype data.

## Stack

- TanStack Start (React 19) + Vite
- TypeScript, Tailwind CSS v4, shadcn/ui, lucide-react
- Recharts for visualizations
- Zod for schema validation

## Getting started

```bash
npm install
npm run dev      # local dev server
npm run build    # production build
npm run lint     # lint
```

## Project layout

- `src/routes/` — pages (TanStack file-based routing); the nine credit-officer screens
- `src/components/healthlens/` — domain components (queue, health card, decision, explainability, fraud, CAM, governance)
- `src/components/ui/` — shadcn primitives
- `src/lib/types.ts` — the frozen `MsmeCase` data contract (engine output shape)
- `src/lib/data/` — deterministic synthetic data engine (archetypes, generator, dataset; leakage-free)
- `src/lib/scoring/` — features, rule-based scorecard, transparent ML viability model, decisioning / BRE
- `src/lib/connectors/` — Account Aggregator / GSTN / UPI / EPFO / DISCOM power / bureau connector stubs
- `src/lib/api/` — ULI / OCEN-style scoring endpoints (Zod-validated)
- `docs/` — Track 03 problem statement and the generated ML model card

## Deployment

`npm run build` produces a Vite + Nitro build in `.output/` (verified working, ~0.3s).

### Cloudflare (default, verified green)

The default build targets Cloudflare Workers via Nitro and works out of the box. After
`npm run build`, deploy with `npx wrangler deploy`. **This is the recommended demo target.**

### AWS / Node / Docker (bank-sandbox target)

The server is a portable Nitro handler. Set `NITRO_PRESET=node-server` (or `aws-lambda`) before
`npm run build` to emit a Node/Lambda bundle you can run on **AWS** (Lambda + CloudFront, Amplify,
or ECS/Fargate via Docker). All prototype data is synthetic and in-process, so no managed
datastore is required to run the demo.

### Vercel (optional)

`vercel.json` sets `NITRO_PRESET=vercel`. Note: the bundled Nitro 3.0-beta has a known upstream
`@vercel/nft` dependency-tracing issue that can break the serverless build — prefer the Cloudflare
or Node/AWS paths above for a guaranteed-green deploy.

### Cloud portability

HealthLens is cloud-agnostic: the frontend compiles to static client assets and the server is a
portable Nitro handler with presets for AWS Lambda, Node, Azure, Netlify, and Cloudflare — the
same codebase deploys to any of them with only a preset change.
