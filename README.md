# IDBI MSME HealthLens

Bank-grade internal web app for IDBI credit officers — an explainable **MSME Financial Health Card** built from consented alternate data (GST, Account Aggregator bank, UPI, EPFO, **electricity / power consumption**, bureau-lite). Built for IDBI Innovate 2026, Track 03 (Financial Inclusion / Digital Lending / Credit Decisioning).

> HealthLens is a decision-support and configurable recommendation engine. IDBI remains the regulated entity and final decision owner. All data shown is synthetic prototype data.

## Stack

- TanStack Start (React 19) + Vite
- TypeScript, Tailwind CSS v4, shadcn/ui, lucide-react
- Recharts for visualizations
- Zod for schema validation

## Getting started

```bash
npm install
npm run dev      # local dev server
npm run build    # production build
npm run lint     # lint
```

## Project layout

- `src/routes/` — pages (TanStack file-based routing); the nine credit-officer screens
- `src/components/healthlens/` — domain components (queue, health card, decision, explainability, fraud, CAM, governance)
- `src/components/ui/` — shadcn primitives
- `src/lib/types.ts` — the frozen `MsmeCase` data contract (engine output shape)
- `src/lib/data/` — deterministic synthetic data engine (archetypes, generator, dataset; leakage-free)
- `src/lib/scoring/` — features, rule-based scorecard, transparent ML viability model, decisioning / BRE
- `src/lib/connectors/` — Account Aggregator / GSTN / UPI / EPFO / DISCOM power / bureau connector stubs
- `src/lib/api/` — ULI / OCEN-style scoring endpoints (Zod-validated)
- `docs/` — product plan, research, implementation plan, model card, AMA transcripts

## Deployment

`npm run build` produces a Vite + Nitro build in `.output/` (verified working).

### Vercel

Preconfigured via `vercel.json` (`NITRO_PRESET=vercel`). In the Vercel project set Framework Preset to **Other** and Build Command to `npm run build`; Nitro emits the Vercel Build Output.

> Heads-up: the bundled Nitro 3.0-beta currently has an upstream `@vercel/nft` dependency-tracing issue that can break the serverless build. If you hit it, pin `nitro` to a stable release, or use the Cloudflare path below (works out of the box).

### Cloudflare (default, verified)

The default build targets Cloudflare Workers via Nitro. After `npm run build`, deploy with `npx wrangler deploy`.

### Cloud portability (AWS and others)

HealthLens is cloud-agnostic: the frontend compiles to static client assets and the server is a portable Nitro handler. Nitro ships presets for AWS Lambda, Azure, Netlify, and Node/Docker, so the same codebase deploys to **AWS** (e.g. Lambda + CloudFront / Amplify) with only a preset change. All prototype data is synthetic and in-process, so no managed datastore is required to run the demo.
